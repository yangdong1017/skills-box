# 二、产品推荐决策树

> 根据客户特征，按以下逻辑判断推荐产品。严格遵循，不要凭感觉。

## 常态化首推决策

```
IF 新客户（从未付费）:
  │
  ├─ IF 无产品 + 无方向 + 预算有限
  │   → 推【渡鸦社群 588】
  │   → 理由："先进社群找方向，看同行怎么做，1v1诊断帮你梳理"
  │   → 不要推训练营（没方向报训练营是浪费钱）
  │
  ├─ IF 无产品 + 无方向 + 预算充足
  │   → 推【渡鸦社群 588】
  │   → 铺垫："等青云计划的项目类训练营出来，第一时间通知你"
  │   → 不要因为预算充足就推高价品，方向没定推什么都是错的
  │
  ├─ IF 有产品/服务 + 想做闲鱼获客
  │   → 推【闲鱼训练营 1980】
  │   → 前提确认："你有自己的后端产品对吧？"
  │   → 如果没有后端 → 退回推588或等项目类训练营
  │
  ├─ IF 有产品/服务 + 想做小红书/全域获客
  │   → 推【渡鸦社群 588】先入门
  │   → 或直接推【墨界系统 5580+】（如果有团队、有矩阵基础）
  │
  ├─ IF 有团队 + 已在做矩阵 + 管理混乱/效率低
  │   → 推【墨界系统】会员¥5580 / 非会员¥7580
  │   → 非会员可建议先入社群588拿会员价，总价588+5580=6168 < 7580
  │   → 核心卖点：系统化管理、8万字定制方案、小群指导
  │
  └─ IF 想用AI工具（墨鸦Claw）
      → 不单独推月卡，推【渡鸦社群 588】
      → 理由："入社群直接送1个月墨鸦Claw + 算力 + 3天训练营"
      → 比单买月卡188划算得多
```

# 五、特殊客户类型处理

## 转介绍人（如本次实战案例）

**特征：** 老会员、推荐过多人、信任度极高、当合作伙伴对待

**处理原则：**
- 不需要建信任，直接聊需求
- 尊重她的判断（她比你更了解她推荐的人）
- 实话实说，不夸大产品（她推荐了人，你夸大了交付，她背锅）
- 让她知道转介绍有返利，维护长期推荐关系
- 主动要求直接对接被推荐人，提高效率

## 老会员续费

**处理原则：**
- 不需要重新讲产品价值
- 重点问："这一年你最大的收获是什么？有什么还没解决的？"
- 根据回答推荐升级路径（年卡/训练营/墨界）
- 续费本身要提前1个月开始跟进

## 同行探底识别

**典型信号：**
- 问"你们服务多久"、"现在有多少会员"
- 问"软件批量拿多少钱"
- 朋友圈有同类产品的推广内容

**处理方式：**
1. 先看朋友圈验证
2. 如果确认是同行 → 正常回答基本信息，不透露核心策略
3. 有些同行会伪装成用户 → 保持警惕但不要误伤真实用户
4. 不需要对抗，保持专业即可
