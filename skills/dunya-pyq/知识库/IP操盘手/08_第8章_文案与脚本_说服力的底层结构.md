# 第8章 文案与脚本：说服力的底层结构

## 开篇故事：一个文案赋予的生命

2018年，一位90后创业者叫张浩，做的是一款冥想软件。产品没问题，界面也漂亮，但首月仅有82个下载。

他找来一个文案老手帮忙改文案。这个老手用了一个标题：

**「每晚10分钟，让你的大脑停止打转」**

就这一句话，仅改这一句，下个月下载量跳到了1.2万。

后来张浩问他怎么做到的。那个文案老手说了一句话：「我没有改产品，只是改了人们看待这个产品的角度。原来的文案在讲功能，而我讲的是他们的痛点。」

这一刻，张浩才明白了一个道理：**好文案不是为了说，而是为了让人听进去。**

本章要讲的，就是让人听进去的底层结构——那些看不见的规律，却能决定内容能否击中人心。

## 8.1 三种注意力钩子：欲望、焦虑、好奇心

你的内容，正在和数百万条信息竞争用户的注意力。

一个普通人每天接收的信息量约140万条，但真正能进入意识的只有12条。这12条，往往来自一个简单的触发——一个钩子。

三种钩子最有效。

### 欲望钩子：我要变成更好的自己

欲望钩子直指人的内在渴望。不是"有需求"，而是"我想成为什么样的人"。

例子很常见：
- 「拍出电影感的短视频」（成为创意人）
- 「10周变身衣架子身材」（成为自律者）
- 「用100块开始月入10万的副业」（成为富有者）

注意，这些不是在卖产品，而是在卖一个身份。人们买的不是课程，而是"我想变成学这个课的那种人"。

欲望钩子的秘诀是**具体化**。「变优秀」太虚。「掌握5个AI工具，让你的工作效率提升300%」就是具体的身份承诺。

建立欲望钩子的三个步骤：

1. **识别目标身份** ——你的用户想成为什么人？
2. **具体化收益** ——这个身份能做什么？
3. **制造对比** ——现在的他和那个他，差距在哪？

表格：欲望钩子的三层递进

| 层级 | 表现形式 | 例子 |
|------|--------|------|
| 第一层 | 功能性承诺 | 「学会剪辑视频」 |
| 第二层 | 身份承诺 | 「成为视频创作者」 |
| 第三层 | 生活方式 | 「用内容赚取自由的生活」 |

越往下走，钩子越有力。

### 焦虑钩子：不做就会失去

焦虑钩子激发的不是"我想拥有"，而是"我害怕失去"。心理学有个概念叫"损失厌恶"，意思是失去100块的痛苦，大于获得100块的快乐。

所以焦虑钩子常常这样用：
- 「如果不会用AI，三年后你会被淘汰」
- 「95%的企业正在流失客户，因为他们还在用过时的文案模板」
- 「你的对手已经在用这个方法抢你的生意」

焦虑钩子很容易被用坏。过度夸大风险，就变成了震惊文案，让人反感。好的焦虑钩子要满足两个条件：

1. **风险必须真实存在** ——不能凭空捏造
2. **损失必须可以量化** ——「可能被淘汰」不如「年损失50万」具体

有效的焦虑钩子是这样的：「B站创意区300个up主已经月入5万，而你还在为稿费烦恼。」

这不是在吓你，而是在用对比创造紧迫感。

### 好奇心钩子：只要你想看，我就有悬念

好奇心是人的天性。打破常识、反转预期、留下悬念，都能激发好奇。

"反常识"是最常见的好奇钩子：
- 「做内容的人都在犯一个错误，就是……」
- 「大多数人都在做的这件事，其实会让你越来越穷」
- 「你以为的"努力"，可能正在毁掉你的事业」

关键是第一句话就要打破预期。读者原本以为要听一个故事，结果你给他一个反转。他的大脑会自动要求看完，这样才能理解。

好奇心钩子的三种形式：

1. **数字悬念** ——「实际上只有3%的人知道……」
2. **观点反转** ——「你一直被骗，真相其实是……」
3. **过程悬念** ——「他是怎么在一个月内做到的？看完这个视频你就知道」

一个容易忽视的细节是：好奇心钩子最容易"烂尾"。你吊足了胃口，但最后的解释非常平凡，用户就会很失望。所以**好奇心钩子只能用于真的有干货的内容**。

## 8.2 故事的七个要素与八个经典母题

为什么有些内容，人们会主动转发？为什么有些故事，被讲过一千遍还有人爱听？

因为背后有结构。

### 故事的七个要素

一个能打动人的故事，必须包含这七个要素：

**1. 主人公**

必须是具体的人。不是"一个年轻女性"，而是"月薪3000的编辑，30岁时突然失业"。细节越具体，代入感越强。

**2. 清晰的处境**

主人公现在的状况如何？最重要的是，这个状况要和你的听众相关。如果讲一个富二代的故事给打工人听，代入感就弱。

**3. 触发的冲突**

一个改变。可能是：
- 外部事件（失业、破产、被骗）
- 内在觉醒（突然明白了什么）
- 新的机会（遇见一个人、发现一个方法）

冲突不是要很戏剧化，只要足以打破日常就够了。

**4. 尝试与阻碍**

主人公开始改变，但遇到了困难。这一段很重要，因为正是这里让故事变得可信。如果一个人遇到困难就顺利解决了，故事就太假。

让我给你一个对比：

- **平凡版** ：「他学习了新技能，然后成功了。」
- **有说服力版本** ：「他报名了课程，第一周完全听不懂。他想放弃，但坚持了下来。第三周他做出了第一个项目，没想到竟然有人愿意付费。」

第二个版本有起伏，有细节，所以更真实。

**5. 关键时刻的抉择**

故事的转折点。往往这时候，主人公要做一个艰难的选择。选择本身不一定有多戏剧化，但这是"人性"最闪耀的时刻。

选择代表什么？代表价值观。所以这一段对应的，往往是你想传达的核心价值。

**6. 结果与收获**

不一定要"大成功"。可以是小的改变，一个新的认知，一个新的可能性。但一定要具体。「他改变了」不如「他月入从3000变成了8000，更重要的是，他不再看老板脸色」。

**7. 对听众的呼应**

这是故事的最后一步——让听众看到自己。这可以是一个问题、一个洞察，或一个暗示：「你现在的情况，和他当时一样吗？」

### 故事框架

用七个要素构建的故事，其实遵循这样的框架：

```
日常 ──[触发]──> 混乱 ──[抉择]──> 重生 ──[回响]──> 启示
```

这个过程中，每一段都要有具体的细节。细节是故事的肉。

### 八个经典故事母题

很多人以为好故事需要完全原创，其实不然。最打动人的故事，往往遵循某些经典的"母题"。用母题不是抄袭，而是用更深层的结构让故事更有力。

**母题1：重生**

一个人，在困顿中找到了新的可能。这是最常见的商业故事母题。核心在于"转折"。

例子：失业设计师自学编程，半年后拿到25万年薪的offer。

**母题2：对抗**

一个人与既定的规则对抗，最后改变了什么。这类故事很有张力。

例子：小镇年轻人对抗传统，用短视频让全国都知道他的品牌，赚回了一套房。

**母题3：发现**

一个人找到了被忽视的真相。这类故事往往能产生"啊，原来如此"的共鸣。

例子：两年研究后发现，阻碍99%人成功的，不是执行力，而是选择错了方向。

**母题4：牺牲**

一个人为了什么放弃了什么。这类故事很容易激发感情共鸣。但注意，在商业内容中要谨慎使用，因为"牺牲"听起来不像成功。

更好的用法是"取舍"：一个创业者放弃了稳定工作，换来了时间自由。

**母题5：成长**

一个人从无知变成了有知。这是最朴素的母题，却最有说服力，因为它代表的是"人可以改变"。

例子：完全不懂运营的上班族，通过系统学习，把自己的小店变成了月入8万的生意。

**母题6：联盟**

一个人找到了志同道合的人，一起做成了什么。这类故事天生有温暖感。

例子：三个不同背景的创意人聚在一起，用内容为小企业带来了转机。

**母题7：救赎**

一个人走错了路，最后回到了正轨。这类故事容易激发反思。

例子：因为错误的学习方法，浪费了两年，后来才明白什么才是正确的方向。

**母题8：传承**

一个人学到了什么，然后传给下一个人。这类故事能激发希望和使命感。

例子：一个逆袭的运营，现在每年培养100个学生做出自己的IP。

表格：故事母题与适用场景

| 母题 | 最适用的背景 | 传达的价值观 |
|------|-----------|---------|
| 重生 | 失败→成功 | 改变永不晚 |
| 对抗 | 少数派vs规则 | 打破束缚 |
| 发现 | 迷茫→觉悟 | 正确认知 |
| 牺牲/取舍 | 稳定vs自由 | 有舍有得 |
| 成长 | 小白→专家 | 学无止境 |
| 联盟 | 个人→团队 | 力量来自连接 |
| 救赎 | 错误→纠正 | 永不放弃 |
| 传承 | 一代→下一代 | 意义在于传承 |

选择故事母题时，要和你的内容目标对应。如果你的目标是"打动小白用户报名课程"，那么"成长"母题的效果最好。如果你想鼓励人们打破规则做新的事，那就用"对抗"母题。

## 8.3 叙事的基本原则：一条主线，层层推进

写文案有个通病：什么都想说，结果什么都没说清。

好的叙事有一条看不见的主线，就像建筑的骨架，外面的砖石往这根骨架上堆。

### 识别你的主线

主线是什么？是贯穿整个内容的一个"进展"。

一篇文案，可能包含10个事实、5个洞察、20个细节。但如果没有主线，这些信息就像散落的珠子，再漂亮也组不成项链。

主线的形式多种多样：

**时间线** ：「从他开始创业到年入百万，经历了什么」
**问题线** ：「大多数人都问这个问题，答案其实是……」
**感受线** ：「从绝望到希望的转变」
**认知线** ：「从错误认知到正确认知」

例子：

一篇讲"如何做好内容"的文案。

- 如果用"时间线"，主线就是：「一个月做出第一条内容→三个月积累粉丝→半年变现」
- 如果用"问题线"，主线就是：「大多数人犯的错误是什么？→为什么会犯这个错误？→怎么避免？」
- 如果用"认知线"，主线就是：「你以为做内容要靠灵感→其实靠的是系统→系统的三个要素是什么」

选哪一条主线？看你的听众最关心什么。新手关心"怎么开始"，所以时间线有效。已经开始做但卡壳的人，关心"我为什么失败"，所以问题线更有力。

### 三层结构的信息组织

确定了主线，怎么组织信息？

用"三层"来思考。

第一层是主线，就像高速公路。
第二层是支撑主线的子论点，像是连接高速公路的城市道路。
第三层是具体细节和例子，像是街道。

一个例子：

**第一层（主线）** ：「内容写得好，取决于一个简单的原理」

**第二层（支撑）** ：
- 原理是什么
- 为什么大多数人没想到
- 怎么用这个原理指导创作

**第三层（细节）** ：
- 一个真实案例
- 一个常见错误的改法
- 一个数据证明
- 一个工具或模板

写的时候，每一层都要清晰。如果一个段落既在讲主线又在讲细节，读者的大脑就要耗电工作，容易疲劳。

### 层层推进的节奏

节奏很重要。

好的节奏是：提出问题→解释为什么→给出方案→证明有效→激励行动。

但不是每一段都要这样。有时候，你可以用"悬念→揭晓"来推进，有时候是"错误→纠正"。

关键是：**每一个自然段都要推进故事**。

如果一段话读完了，读者的认知没有推进，就是冗余。

表格：常见的推进方式

| 推进方式 | 结构 | 适用场景 |
|--------|------|--------|
| 悬念型 | 问题→答案 | 吸引注意力 |
| 递进型 | 现象→原因→结果 | 建立认知 |
| 对比型 | 错误→正确 | 纠正观念 |
| 证明型 | 观点→案例→数据 | 增强说服力 |
| 情感型 | 场景→感受→启示 | 建立共鸣 |

一篇好的文案，往往混合使用这些推进方式。开头用"悬念型"吸引注意，中间用"递进型"和"证明型"建立信服力，结尾用"情感型"或"行动型"激励行动。

## 8.4 标题的四层结构与六大常用套路

一个标题可以决定99%的人是否看这篇内容。

绝对不是夸张。

### 标题的四层结构

好标题往往有四层：

**第一层：钩子**

这是吸引注意的部分。必须打破预期，或触发三种注意力钩子之一。

例子：
- 「你的好友正在流失，而你还不知道原因」（焦虑）
- 「他靠这个副业赚了50万」（欲望）
- 「明星都在用的这个方法，你可能没听过」（好奇）

**第二层：承诺**

钩子吸引注意后，要快速让人知道"这篇内容对我有什么用"。

例子：
- 「5分钟学会这个技巧……」
- 「赚钱的秘密其实是……」
- 「为什么你的内容没人看，答案在这里」

**第三层：具体性**

要让人感到这不是鸡汤，而是真有方法。通常用数字、时间或具体名词。

- 「用3个工具」不如「用Figma、ChatGPT和剪映」
- 「快速变现」不如「30天月入5000」

**第四层：好奇心的尾巴**

留一点悬念，让人不得不点开看。

例子：
- 「……这件事99%的人都做反了」
- 「……你的竞争对手已经在用了」
- 「……就这一步，我从月入3千变成3万」

这四层合在一起，一个标题就成立了：

「你还在用老套的涨粉方法？这3个工具的配合，让我30天粉丝翻倍，而且……」

### 六大常用套路

不是说标题有公式就一定有效。但掌握常见套路，能大幅提高成功率。

**套路1：数字+承诺**

「3个习惯让你月入10万」
「90%的人不知道的赚钱秘诀」
「用5分钟学会这个……」

为什么有效？因为数字给了人具体的预期。「让你赚钱」太虚，「让你月入10万」就能在脑子里形成画面。

**套路2：问题+反转**

「你以为靠执行力成功？错了，真正的原因是……」
「为什么他一个人能做出别人团队的成果？」
「做内容最常见的错误，你中了几个？」

为什么有效？因为打破了预期。人的大脑天生对"反常"敏感。

**套路3：情感+结果**

「从绝望到月入10万，他只做了这一件事」
「曾经月薪3000，现在他的课程卖出了100万」
「被骗了3年，他终于明白了……」

为什么有效？因为有起伏和对比。情感让人产生共鸣，结果让人看到希望。

**套路4：对比+时间**

「3年前他一无所有，现在是行业头部创作者」
「从学员到老师，只用了6个月」
「等了两年才明白的事，你可以今天就知道」

为什么有效？对比显示变化，时间让变化显得可能和可控。

**套路5：权威+独家**

「这个方法，只有前100个学员知道」
「头部创意人用的这个技巧，终于要公开了」
「被隐瞒的真相，今天我全部说出来」

为什么有效？稀缺性激发欲望。人们天生觉得"只有我知道的"才是宝贝。

**套路6：矛盾+解释**

「花了10倍的时间，赚钱反而更快了」
「不努力反而成功了，原因竟然是……」
「越放松越有效率，这个悖论怎么破？」

为什么有效？矛盾激发好奇心。大脑会自动要求解决矛盾。

### 标题写作的四个禁忌

**禁忌1：标题超过15个字**

太长了，人不会完整读。在手机上尤其如此。应该用副标题补充信息。

**禁忌2：标题太虚**

「改变你的人生」「成为你想成为的人」这类话能触发的只有共鸣，不能触发行动。加入具体数字或场景。

**禁忌3：过度承诺**

「99%的人都赚不到钱，但用这个方法……」然后给的方法不值99%的期待，就是欺骗。标题承诺了什么，内容就要兑现什么。

**禁忌4：标题和内容不符**

这是最致命的。因为这样做不仅一次浪费人的时间，还会摧毁信任。一旦信任摧毁，再好的内容也救不了。

## 8.5 冲突与悬念：让用户看下去的引擎

有个观察很有意思：最无聊的内容，往往是"从头到尾都很和谐"的内容。

而最能吸引人的，都有某种形式的冲突。

### 冲突的三个层次

**第一层：概念冲突**

两个相反或对立的观点碰撞。

例子：
- 「大多数人说靠努力成功，但数据显示……」
- 「你学了所有的方法，但还是失败了，为什么？」
- 「越来越多的人在做副业，可更多的人在赔钱」

这类冲突最轻，但很有效。它能让人产生"想搞清楚"的欲望。

**第二层：角色冲突**

一个人和另一个人、或一个人和环境的对抗。

例子：
- 「他和所有人都说他疯了，但他还是坚持创业」
- 「大公司的HR告诉她，你的想法不切实际，但她还是做了」

这类冲突有戏剧感。故事因此而产生。

**第三层：自我冲突**

一个人内心的矛盾。这是最深的冲突。

例子：
- 「她想辞职创业，但害怕失败」
- 「他赚到了钱，却越来越空虚」
- 「成功了，但失去了初心」

自我冲突最能打动人，因为每个人都有内在的矛盾。当你把一个人的内在矛盾展现出来，人们就会看到自己。

### 悬念的机制

悬念不是"故意卖关子"。好的悬念是：读者有了某个问题，非常想知道答案。

悬念的产生需要三个条件：

**条件1：一个清晰的问题或缺口**

「这件事怎么会发生？」「下一步会怎样？」「答案是什么？」

不能是模糊的。「大家都很好奇」不够，要让他们具体地想知道什么。

**条件2：足够的重要性**

这个问题对他们要重要。如果问题太小，即使有悬念也没人关心。

**条件3：延迟满足**

不是立刻回答。而是在回答前，让他们多想一想，或再等一等。

例子来看清楚这个机制：

```
不好的悬念：
"我要告诉你一个秘密……（卖关子1分钟后）……我只用了这一个方法。"

好的悬念：
"你知道为什么有的人内容一发就是爆款，有的人发了三年还没粉丝吗？
我曾经也很困惑。后来我采访了50个头部创作者，发现他们都有一个共同点。
这个共同点不是才华，也不是运气……"
```

第一个是强行延迟，第二个是自然的逻辑延迟。读者在想"那个共同点是什么"的时候，就已经陷入悬念。

### 建造悬念的四个技巧

**技巧1：部分展示**

不要一下子全部说出。先说一半，让他猜另一半。

例子：「他之所以能月入10万，不是因为他有多厉害，而是因为……」

**技巧2：反复强调重要性**

「这一点特别重要」「大多数人都忽视了这个」「99%的人因为不知道这个失败了」

当你反复强调重要性，读者就会觉得"我一定要知道答案"。

**技巧3：预埋线索**

提前提到一些相关但不直接的信息，让读者产生"为什么要提这个"的疑问。

**技巧4：对比制造反差**

「看起来不起眼的这一点，却改变了一切」
「最简单的方法，反而最有效」

反差越大，悬念越强。

## 8.6 金句的四重作用与六种制造方法

什么叫金句？

就是那种读完以后，人会下意识把它记住、想分享的句子。

金句有四重作用。

### 金句的四重作用

**作用1：记忆点**

好的金句容易被记住。「我没有改产品，只是改了人们看待这个产品的角度」这句话，人们会反复回想。

而"好产品需要好文案"这句话，人们就很容易忘。

为什么？因为金句往往有某种形式的美感——韵律、对称、反转。

**作用2：传播点**

容易记住的东西，就容易被传播。有人在朋友圈转发金句，就是在帮你传播。

**作用3：信任点**

一个金句，往往代表了你对某件事的深刻理解。人们通过金句感受到你的专业性。

**作用4：文案的灵魂**

整篇文案可能有1000字，但金句是那1-2句，代表整个内容的核心价值。

### 六种制造金句的方法

**方法1：对称法**

用相同的句式结构，产生美感和力量。

例子：
- 「选择不对，努力白费」
- 「他越来越成功，我越来越迷茫」
- 「不是你不努力，而是努力的方向不对」

对称法的关键是前半句和后半句的重量要差不多。

模板：「不是……，而是……」
「……越……，……越……」
「……不……，……」

**方法2：反转法**

打破读者的预期，得出相反的结论。

例子：
- 「内向的人更容易成功，因为他们听得比说得多」
- 「越想赚钱越赚不到，因为焦虑会影响判断」
- 「放弃比坚持需要更大的勇气」

反转法要注意：反转本身要合理，而不是为了反转而反转。

**方法3：比喻法**

用一个具体的比喻，让抽象的东西变得具体。

例子：
- 「好的文案就像一颗钉子，要一击即中」
- 「内容是种子，运营是浇水，变现是收获」
- 「算法就像一个脾气怪的评委，你要摸清他的习惯」

比喻法的秘诀是找到核心的相似性。不是所有的比喻都成立。

**方法4：递进法**

从浅层到深层，逐步深化观点。

例子：
- 「知道方法不等于做对方法，做对方法不等于坚持，坚持不等于看到成果」

这种金句看起来简单，但传达的是一个完整的认知升级过程。

**方法5：数字法**

用数字来强调对比或力量。

例子：
- 「90%的失败来自于前10%的决定」
- 「用20%的精力，做80%的重要事」
- 「一个小错误，价值1000个改正」

数字法要注意真实性。如果数字是随便编的，一旦被识别出来，就会损害信任。最好用真实数据或明显的比例。

**方法6：悖论法**

陈述一个看起来矛盾的观点，然后用逻辑解释为什么它是对的。

例子：
- 「越想成功越失败，因为成功恰好来自于不害怕失败」
- 「花时间看起来浪费时间，其实是最高效的投资」
- 「越强调自己有本事，越容易失去机会」

这是最高级的金句方式，因为它让人需要思考。但也最容易失效——如果解释不好，就变成了鸡汤。

### 金句的使用位置

金句不是随便放的。位置很重要。

- **开头金句** ：引入话题，吸引注意
- **转折点金句** ：总结前面的论述，为下一段铺垫
- **结尾金句** ：升华主题，让人记住

一篇文案通常需要3-5个金句。太多反而显得没有重点，太少容易显得平淡。

## 8.7 六大脚本类型的标准化生产

从开始做内容到现在，我见过无数人的困扰是："我不知道怎么写脚本。"

其实脚本没那么神秘。就是把你想说的话，按照一定的格式和节奏组织起来。

### 六大脚本类型

**类型1：问题解决型脚本**

这是最常见的类型。你发现了一个问题，然后给出解决方案。

结构：
```
1. 打开问题（让人发现自己有这个问题）
2. 分析问题（为什么会有这个问题）
3. 展示后果（不解决会怎样）
4. 给出方案（3-5个步骤）
5. 验证效果（案例或数据）
6. 行动号召（现在就开始）
```

例子脚本框架：
```
开头：你有没有发现，做了一个月的内容，粉丝才增长100个？
原因：这不是你的错，是因为……（解释算法、方向、执行等）
后果：如果一直这样，你会……
方案：改变这三个地方：
      第一个是……
      第二个是……
      第三个是……
验证：用这个方法的人，都在……
行动：从今天开始……
```

**类型2：故事演绎型脚本**

以一个故事为主，故事中自然呈现观点。

结构：
```
1. 开场故事（一个具体的人和事件）
2. 故事发展（冲突和转折）
3. 关键时刻（主人公的选择或顿悟）
4. 结果（变化和收获）
5. 洞察提取（从故事中得到什么认知）
6. 对听众的启示（这对你意味着什么）
```

这类脚本最容易产生共鸣，因为人们更容易代入故事。

**类型3：对比分析型脚本**

通过对比来阐述观点。对比可以是"之前vs现在"，也可以是"他们vs我们"。

结构：
```
1. 呈现现象（大多数人的做法）
2. 分析其局限（为什么这样有问题）
3. 展示对立面（正确的做法是什么）
4. 分析优势（为什么这样更好）
5. 提供工具或步骤（怎么做）
6. 鼓励改变（是时候转变了）
```

**类型4：干货输出型脚本**

直接输出知识或方法。没有太多故事，重点是有用。

结构：
```
1. 承诺（这个内容能给你什么）
2. 核心观点（最重要的3-5个点）
3. 每个点的展开（细节、例子、为什么）
4. 总结（重新强调核心点）
5. 应用（怎么用这些知识）
6. 进阶（接下来的方向）
```

这类脚本对已经有基础的观众有效。他们来就是要学东西。

**类型5：观点表达型脚本**

表达一个观点或立场。通常带有一定的观点冲突。

结构：
```
1. 陈述观点（我认为……）
2. 为什么这样想（原因1、原因2）
3. 常见反对意见（有人会说……）
4. 回应反对（但实际上……）
5. 展示证据（案例、数据）
6. 行动建议（基于这个观点，应该……）
```

这类脚本能打造个人立场，但需要谨慎，因为容易树立对立面。

**类型6：启发式脚本**

不是直接给答案，而是引导人思考。这类脚本往往能产生最深的印象。

结构：
```
1. 抛出问题（一个值得思考的问题）
2. 大多数人的答案（通常是浅层的）
3. 深层思考（换个角度看会怎样）
4. 呈现悖论（为什么直觉答案不对）
5. 提示新角度（试试这样想……）
6. 留给听众的思考空间（你认为呢？）
```

### 脚本的标准化模板

为了提高效率，好多人会用模板。模板本身没错，但要避免套到没有生气。

关键是：**模板是骨架，内容是血肉**。

一个标准化的视频脚本应该包含：

```
【视频标题】
【视频时长目标】 如：6分钟
【目标观众】
【核心承诺】 看完这个视频，能得到什么

【开头（0-30秒）】
- 钩子
- 承诺

【主体（第一部分）】
- 观点1
  └─ 支撑论点
  └─ 案例
- 观点2
  └─ 支撑论点
  └─ 案例
- 观点3
  └─ 支撑论点
  └─ 案例

【主体（第二部分）】
- 应用或深化

【结尾（最后15秒）】
- 金句总结
- 行动号召
- 预告下一期内容
```

一个例子脚本（5分钟视频）：

```
【标题】为什么你的内容没人看？真正的原因在这里

【开头 0-30秒】
"你一个月发10条内容，点赞破百吗？
如果没有，我告诉你，这不是内容的问题。
这是一个系统问题。"

【主体 30秒-4分钟】
"大多数人犯的第一个错误：只想着'我要说什么'

（案例：小王发了一条关于时间管理的tips，5个赞）

但真正看内容的人在想什么？
'这对我有什么用？'

所以我们要反过来想：先想'他们想听什么'，再决定'我说什么'"

（演示：同样的内容，换一个角度讲，从5赞到500赞）

"第二个错误：开头没有钩子

一个好的开头，要在3秒内……"

（举例2-3个）

"第三个错误：节奏太平，没有起伏……"

【结尾 4-5分钟】
"所以记住这三个：想清楚他们的真实需求，
用钩子打开他们的注意，
用节奏保住他们的注意。
坚持这个系统，内容迟早会爆。

下期我们讲怎么找对自己的账号定位。点个关注不迷路。"
```

### 脚本写作的三个常见错误

**错误1：写成了广告稿**

脚本不是卖东西的稿子。如果全是在推销，人们会很反感。即使没有产品要卖，如果脚本的目的太功利，观众也能感受到。

正确的做法：先给价值，再有任何商业目的。比例应该是 80% 价值 + 20% 目的。

**错误2：信息密度太大**

新手容易犯这个错。一个5分钟的视频，硬是塞进了20个观点。结果观众什么也记不住。

正确的做法：一个视频讲一个主题，最多三个子观点。宁可浅而全，也别贪心。

**错误3：没有清晰的行动指引**

脚本最后要明确地告诉人们"现在该做什么"。不是"谢谢观看"就完了。

可以是：
- 「现在就开始做这个」
- 「在评论区告诉我你的想法」
- 「点击链接看详细教程」
- 「下周同样的时间，我们讨论进阶版」

## 本章小结

本章讲的是说服力的底层结构。

重点不在于学会某一种写法，而是理解：**为什么有的话能打动人，有的话如同废话。**

核心观点有四个：

1. **注意力是稀缺资源**，三种钩子（欲望、焦虑、好奇）是打开注意力的钥匙。选哪一把钥匙，要看听众最关心什么。

2. **故事是最有说服力的形式**。不是因为故事优美，而是因为故事能让人代入，让人感受，让人记住。每个故事都遵循某种母题，学会这些母题，你就有了无穷的素材库。

3. **好的叙事有一条看不见的骨架**。这条骨架保证了信息的流向和节奏。而细节是血肉。优秀的文案，就是精心选择的细节，围绕这条骨架堆积。

4. **金句是文案的灵魂**。它们让人记住，让人传播，让人信任。而金句的制造不是灵感，是方法。你可以学会。

在实践中，这些方法往往混合使用。一篇文案可能先用故事激发共鸣，再用数据增强说服力，最后用一句金句升华。一个视频脚本可能开头用"好奇心钩子"吸引，中间用"对比"纠正观念，结尾用"行动号召"激励改变。

最后一个建议：**不要为了用方法而用方法**。真实感永远是第一位的。如果你的故事是真实的，你的数据是准确的，你的观点是深思熟虑的，那么这些方法只是帮助你更清晰地表达，而不是包装虚假。

相反，当你的内容本身没有价值的时候，再高明的文案技巧也只是临时的说服，最后都会被戳穿。

## 本章工具箱

**工具1：标题评测表**

写完标题，用这个表评测一下：

| 评测项 | 是否满足 | 说明 |
|--------|--------|------|
| 有钩子吗？ | ☐ | 能打破预期或触发三种钩子之一 |
| 有承诺吗？ | ☐ | 清晰说明价值或结果 |
| 有具体性吗？ | ☐ | 用了数字、时间或具体名词 |
| 长度在15字以内吗？ | ☐ | 容易被完整记住和传播 |
| 能自己点开吗？ | ☐ | 如果这是别人的文章，自己会不会点 |

满足4个以上，这个标题就合格了。

**工具2：故事框架填空**

需要讲一个故事时，用这个框架：

```
【主人公】：
【初始状态】：
【触发事件】：
【尝试与困难】：
【关键抉择】：
【最终结果】：
【核心启示】：
【对听众的呼应】：
```

填完这个，你就有了故事的骨架。

**工具3：脚本快速写法**

给5分钟脚本的时间配置：

- 开头钩子：30秒
- 观点1展开：80秒
- 观点2展开：80秒
- 观点3展开：80秒
- 小结和行动号召：30秒

每个观点的展开：观点陈述（10秒）+ 案例或数据（50秒）+ 转折（20秒）

**工具4：金句速成法**

没想到好句子时：

1. 找到你要讲的核心观点
2. 用"不是……而是……"重述一遍
3. 如果觉得平淡，加一个对比或反转
4. 如果还觉得弱，用一个比喻

例如：
- 核心观点：「做好内容的关键是理解观众」
- 重述：「不是内容本身，而是观众的理解」
- 加对比：「你越想证明自己有本事，就越难理解别人的需求」
- 就成了一个有力量的观点

**工具5：内容对标表**

要发内容前，用这个表对标一个做得好的内容：

| 对标项 | 他们怎么做的 | 我可以学什么 |
|--------|-----------|-----------|
| 开头方式 | | |
| 钩子类型 | | |
| 故事or数据 | | |
| 节奏变化 | | |
| 金句数量 | | |
| 结尾方式 | | |

对标不是抄袭，是学习他们的结构逻辑。
