### A.4 SQL必会查询模式速查

#### 数据提取与清洗
```sql
-- 模式1：多表联接与聚合
SELECT a.user_id, COUNT(b.order_id) as order_cnt, SUM(b.amount) as total_amount
FROM users a
LEFT JOIN orders b ON a.user_id = b.user_id
WHERE a.create_date >= '2024-01-01'
GROUP BY a.user_id
HAVING COUNT(b.order_id) > 0;

-- 模式2：时间序列分析
SELECT DATE(order_date) as date, COUNT(*) as order_cnt, SUM(amount) as daily_amount
FROM orders
WHERE order_date BETWEEN '2024-01-01' AND '2024-12-31'
GROUP BY DATE(order_date)
ORDER BY date;

-- 模式3：同比环比分析
SELECT
  DATE_FORMAT(order_date,'%Y%m') as month,
  COUNT(*) as order_cnt,
  LAG(COUNT(*)) OVER (ORDER BY DATE_FORMAT(order_date,'%Y%m')) as last_month_cnt,
  ROUND((COUNT(*) - LAG(COUNT(*)) OVER (ORDER BY DATE_FORMAT(order_date,'%Y%m')))
    / LAG(COUNT(*)) OVER (ORDER BY DATE_FORMAT(order_date,'%Y%m')) * 100, 2) as growth_rate
FROM orders
GROUP BY DATE_FORMAT(order_date,'%Y%m');

-- 模式4：分层分析（RFM模型）
SELECT
  user_id,
  DATEDIFF(NOW(), MAX(order_date)) as recency,
  COUNT(*) as frequency,
  SUM(amount) as monetary,
  CASE WHEN DATEDIFF(NOW(), MAX(order_date)) <= 30 THEN 'R1' ELSE 'R2' END as R_segment,
  CASE WHEN COUNT(*) >= 5 THEN 'F1' ELSE 'F2' END as F_segment
FROM orders
GROUP BY user_id;

-- 模式5：漏斗分析（转化路径）
SELECT
  step,
  COUNT(DISTINCT user_id) as user_cnt,
  LAG(COUNT(DISTINCT user_id)) OVER (ORDER BY step) as prev_cnt,
  ROUND(COUNT(DISTINCT user_id) / LAG(COUNT(DISTINCT user_id)) OVER (ORDER BY step) * 100, 2) as conversion_rate
FROM (
  SELECT user_id, 1 as step FROM events WHERE event_type = 'page_view'
  UNION ALL
  SELECT user_id, 2 as step FROM events WHERE event_type = 'add_to_cart'
  UNION ALL
  SELECT user_id, 3 as step FROM events WHERE event_type = 'checkout'
) t
GROUP BY step
ORDER BY step;
```

| 函数 | 用途 | 语法 |
|------|------|------|
| `ROW_NUMBER/RANK` | 排名 | `ROW_NUMBER() OVER (ORDER BY amount DESC)` |
| `LAG/LEAD` | 前后对比 | `LAG(amount, 1) OVER (ORDER BY date)` |
| `SUM...OVER` | 运行总计 | `SUM(amount) OVER (ORDER BY date)` |
| `CASE WHEN` | 条件分组 | `CASE WHEN amount > 100 THEN '高' END` |

### A.6 BI工具选择指南

#### Tableau vs PowerBI对比

| 维度 | Tableau | PowerBI |
|------|---------|---------|
| **价格** | 较贵（$70/月起） | 相对便宜（$10-20/月） |
| **易用性** | 需要学习曲线 | 对Office用户友好 |
| **数据源** | 丰富（100+） | 主要Excel、SQL |
| **交互性** | 卓越 | 良好 |
| **企业支持** | 强大 | 日趋完善 |
| **适用场景** | 复杂可视化、企业级 | 中小型企业、微软体系 |

#### 选择建议

**选Tableau：**
- 需要复杂、高度定制的可视化
- 要求与非技术人员深度交互
- 企业已有Tableau部署
- 数据源类型复杂多样

**选PowerBI：**
- 企业主要用Office和Azure生态
- 对成本敏感
- 需要与Excel无缝集成
- 团队规模中等

**选开源方案（Metabase/Superset）：**
- 完全自主控制，无供应商锁定
- SQL基础较好
- 部署在私有服务器
- 定制开发能力强

#### 快速上手Tableau步骤

1. **连接数据**：数据→新数据源→选择数据库
2. **创建维度与度量**：默认识别，可手动调整
3. **拖拽构建图表**：维度拖到行列，度量拖到值
4. **美化与交互**：颜色、大小、提示卡、操作
5. **发布共享**：工作簿→发布→Tableau Server

#### 快速上手PowerBI步骤

1. **导入数据**：获取数据→选择数据源→加载
2. **建立数据模型**：关系视图→创建表关系
3. **构建可视化**：可视化→拖拽字段→选择图表类型
4. **添加交互**：切片器、钻取、导航
5. **发布分享**：发布→选择工作区→共享链接

- 漏斗每层转化率计算
- 关键瓶颈层识别与优化
- 转化率优化的A/B测试框架

**应用方法**：
1. 定义完整的转化漏斗：访问→浏览→加入购物车→下单→支付→完成
2. 计算各层的转化率与累积转化率
3. 对比历史数据，找出异常层级
4. 按用户特征、流量来源分层分析漏斗
5. 对转化率最低的层级进行优化实验

**数据看板指标**：各层转化率、漏斗总体转化率、步长转化率、漏斗流失人数

#### 场景11：产品定价与客单价优化
**关键词**：定价、客单价(ASP)、价格策略、销售额

**相关知识点**：
- 价格敏感度分析
- 不同价格点的销量与收益曲线
- 价格分层与产品组合策略
- 动态定价与促销优化

**应用方法**：
1. 收集不同价格点下的销量数据，计算价格-销量关系
2. 分析不同用户群体的价格敏感度
3. 建立收益优化模型：找出最优价格点 = 最大(价格 × 销量)
4. 设计多层级产品与定价，提升客单价
5. A/B测试不同促销方案的客单价影响

**数据看板指标**：客单价(ASP)、按价格点的销售额分布、销售额、毛利率、转化率

#### 场景12：促销优化与活动效果评估
**关键词**：促销、活动、优惠、ROI

**相关知识点**：
- 促销活动的增量效果评估
- 因果推论与对照组设计
- 不同优惠方式的转化影响
- 活动周期分析与计划

**应用方法**：
1. 建立对照组：有促销vs无促销，控制其他变量
2. 计算促销增量 = 促销期销售 - 预期正常销售
3. 计算活动ROI = (增量收入 - 优惠成本 - 运营成本) / 运营成本
4. 分析不同优惠方式(折扣、满减、赠品)的转化率
5. 识别参加活动的高价值用户，优化运营

**数据看板指标**：活动参与人数、转化率、增量销售额、活动ROI、用户价值
